    Trustworthy AI

    System Overview

    User Classes

    • Model provider: a provider of the machine learning model.

    • Data provider : a provider of the data used to test the model. These Partners are trusted to the extent that they are expected to provide non-fraudulent data. Data 61 explained that such Partners could be Banks, Universities, or other large institutions. In practice, only vetted entities would be allowed to create an account.

    • Buyer: an entity wanting to buy a model for either personal or commercial use.

    Overview of the system from each user’s perspective

    Our platform is a website. Each class of user will use the website slightly differently.

    • Model provider are the admins of the system. Their accounts have special permissions that allow them to both upload models and then pay Partners for their data. For each model they upload, they also provide the training data, and a set of specifications for test data format. This is so that Partners know how to upload their data, and so that its quality can be assessed by comparing it to the original training data.

    Step 1: Upload a model as an executable file， data specifications given as a JSON file， training data as csv file and label as csv file. Note the label file here will be used as the label when creating confusion matrix.

    step 2) Upload training data as CSV.

    Step 3) Wait for Partners to upload testing data.

    • Data provider have standard accounts. They cannot upload models. However, they can browse through those uploaded by the engineers, and then upload data in order to get rewarded with Ether. Partners can only upload one set of data per model, in order to prevent them from trying to claim multiple payments from the same, or similar, data.

    Step 1: Upload a model as an executable file， data specifications given as a JSON file， training data as csv file and label as csv file. Note the label file here will be used as the label when creating confusion matrix.

    Step 2) Upload data as CSV and see how the model performed.

    Step 3) Get the test result and get paid automatically.

    • Buyers also have standard accounts, with the same permissions as Partners. The only difference between Partners and Buyers is the motivation for each user to use our platform. The latter wishes to claim rewards for their data, while the former wishes to verify the quality of a model by analysing its performance. Buyers cannot purchase models on our system - they will need to contact the Engineers directly. The steps which a Buyer would carry out are the same as that of a Partner, however, they would not upload any data, nor get paid for it.
